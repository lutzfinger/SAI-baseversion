## 2. Gap analysis vs Hermes

Compared SAI to `github.com/nousresearch/hermes-agent` to understand what's missing for a
general-purpose, self-improving agent.

**SAI strengths Hermes lacks:**
- Production-grade audit trail, policy gates, eval-first discipline
- Human-in-the-loop preference capture via Slack
- Hash-verified prompt loading, cascade early-stop, per-workflow OAuth scopes
- Skill plug-in protocol with mandatory eval dataset declarations

**SAI gaps vs Hermes:**

| Gap | What Hermes has |
|---|---|
| Generic multi-turn agent loop | `HermesAgentLoop` with pluggable tool dispatch |
| Sandboxed execution | 7 backends (Docker, Modal, SSH, Vercel, etc.) |
| Benchmark evals | SWE-Bench, TerminalBench2 environments |
| Batch trajectory generation | `batch_runner.py` — parallel rollouts → HuggingFace datasets |
| RL training loop | Atropos integration, PPO/GRPO, On-Policy Distillation |
| Verifiable reward computation | `ToolContext` — reward fn gets same sandbox state the agent used |
| Cross-session memory | Persistent user modeling |
| General-purpose tool registry | 40+ tools with distribution sampling |

**Key insight:** SAI already captures all the right human signals (approvals, Slack feedback,
eval record reality observations) — they just weren't structured as reward scalars or
converted to training data. The RL framework below closes that gap.

## 3. RL-from-human-feedback framework (`app/rl/`)

Built a complete RLHF layer on top of SAI's existing human feedback infrastructure.
No new external dependencies. No changes to existing eval, cascade, or observability code
beyond a two-line hook in the eval agent.

### Architecture

```
Human signals (existing)          RL layer (new)              Export
─────────────────────────         ──────────────────────────  ──────────────────
ApprovalRequest (SQLite)    ──→   HumanRewardScorer      ──→  DPO JSONL
SlackFeedbackRecord (SQLite)──→   ScoredTrajectory        ──→  SFT JSONL (ShareGPT)
EvalRecord (JSONL)          ──→   PreferencePairBuilder   ──→  Scored-PPO JSONL
AgentInvocation (audit)     ──→   RawTrajectory (full)
                                  BatchTrajectoryRunner
```

#### `app/rl/models.py`

Defines the core types:

```python
class RewardSource(StrEnum):
    APPROVAL_APPROVED  = "approval_approved"   # +1.0
    APPROVAL_DENIED    = "approval_denied"     # -1.0
    SLACK_POSITIVE     = "slack_positive"      # +1.0
    SLACK_NEGATIVE     = "slack_negative"      # -1.0
    SLACK_EDIT         = "slack_edit"          # +0.3  right intent, wrong execution
    EVAL_GROUND_TRUTH  = "eval_ground_truth"   # +1.0
    EVAL_MISMATCH      = "eval_mismatch"       # -0.8
    ABSTAIN            = "abstain"             # 0.0   excluded from training

class TrajectoryStep      # one tool call, untruncated args + result
class RewardSignal        # scalar reward from one human feedback event
class ScoredTrajectory    # complete RL training example (trajectory + reward)
class PreferencePair      # one DPO example: (system, prompt, chosen, rejected)
```

#### `app/rl/trajectory.py`

Two append-only JSONL stores mirroring the `EvalRecordStore` pattern:

- **`TrajectoryStore`** — stores `RawTrajectory` (full untruncated agent runs)
  - Partitioned by `workflow_id`: `<root>/<workflow_id>.jsonl`
  - `append()`, `read_all(workflow_id)`, `find_by_invocation_id()`, `link_record()`
  - `link_record()` sets `record_id` post-hoc once the corresponding `EvalRecord` is known

- **`ScoredTrajectoryStore`** — stores `ScoredTrajectory` (rewards attached)
  - Partitioned by `workflow_id`: `<root>/<workflow_id>_scored.jsonl`
  - `read_all_workflows()` reads across all workflow files

Default locations:
```
~/Library/Logs/SAI/trajectories/<workflow_id>.jsonl          # RawTrajectory
~/Library/Application Support/SAI/state/rl/scored/          # ScoredTrajectory
```

#### `app/rl/reward.py`

`HumanRewardScorer` — three independent signal sources:

```python
scorer = HumanRewardScorer()

# From approval gate
sig = scorer.score_from_approval(approval_request, trajectory_id=tid)

# From Slack button / emoji / text reply
sig = scorer.score_from_slack_feedback(feedback_record, trajectory_id=tid)

# From EvalRecord reality observation (only when is_ground_truth=True)
sig = scorer.score_from_eval_record(eval_record, trajectory_id=tid)
```

Slack classification logic:
- `action_id` in `{approve, approved, yes, correct, accept, confirm}` → `SLACK_POSITIVE`
- `action_id` in `{reject, rejected, deny, denied, no, incorrect}` → `SLACK_NEGATIVE`
- `value` is `✅` / `:white_check_mark:` → `SLACK_POSITIVE`
- `value` is `❌` / `:x:` → `SLACK_NEGATIVE`
- `text` contains `{edit, revise, correct, update, change, fix}` → `SLACK_EDIT`
- Anything else → `ABSTAIN` (excluded from training)

#### `app/rl/preference_pairs.py`

`PreferencePairBuilder` — groups `ScoredTrajectory` by `(system_prompt, user_message)` and
pairs the highest-reward trajectory against the lowest-reward one:

```python
builder = PreferencePairBuilder(min_reward_gap=0.5)
pairs = builder.build_pairs(scored_trajectories)
```

Guards:
- `ABSTAIN` trajectories excluded before grouping
- Identical `final_response` strings never paired
- `chosen.reward - rejected.reward < min_reward_gap` skipped (noisy signal)

#### `app/rl/batch_runner.py`

`BatchTrajectoryRunner` — reads historical `EvalRecord`s + `RawTrajectory`s, scores them,
writes `ScoredTrajectory`s. No LLM calls.

```python
runner = BatchTrajectoryRunner(
    eval_store=eval_store,
    trajectory_store=traj_store,
    run_store=run_store,
    scorer=HumanRewardScorer(),
)
summary = runner.run(
    workflow_ids=["sai-email-interaction", "newsletter-identification"],
    output_store=scored_store,
    only_ground_truth=True,   # skip records without confirmed reality
)
print(summary.report())
```

Scoring priority:
1. `EvalRecord.reality` if `is_ground_truth=True`
2. Fallback: `ApprovalRequest` linked via `run_id`
3. If neither → `ABSTAIN` (excluded)

#### `app/rl/exporters.py`

`HuggingFaceExporter` — three output formats:

```python
exporter = HuggingFaceExporter()

# DPO — trl.DPOTrainer format
n = exporter.export_dpo(pairs, Path("out/dpo_pairs.jsonl"))
# → {"system", "prompt", "chosen", "rejected", "metadata"}

# SFT — ShareGPT messages format
n = exporter.export_sft(scored, Path("out/sft.jsonl"), min_reward=0.3)
# → {"system", "messages": [user, tool*, assistant], "metadata"}

# Scored — PPO/GRPO format
n = exporter.export_scored(scored, Path("out/scored_ppo.jsonl"))
# → {"prompt", "response", "reward", "metadata"}
```

`min_reward=0.3` on SFT export means only trajectories with positive human signal are used
as supervised fine-tuning examples.

## 4. CLI query tool (`scripts/sai_rl_report.py`)

Queryable text-mode and JSON report over the RL stores.

```sh
# Full summary
python -m scripts.sai_rl_report

# Filter options
python -m scripts.sai_rl_report --workflow sai-email-interaction
python -m scripts.sai_rl_report --source approval_approved
python -m scripts.sai_rl_report --min-reward 0.0       # positives only
python -m scripts.sai_rl_report --max-reward 0.0       # negatives only

# Inspection
python -m scripts.sai_rl_report --pairs --gap 0.5      # show DPO pair detail
python -m scripts.sai_rl_report --tail 10              # last 10 scored

# Machine-readable
python -m scripts.sai_rl_report --json
python -m scripts.sai_rl_report --json | jq '.trajectories[] | {id: .trajectory_id, reward}'
```

Report sections: scored count, raw count, pair count, reward distribution (ASCII bar),
by reward source, by workflow, by model, total inference cost, optional tail, optional
pair detail.

Custom store roots: `--scored-root <path>` and `--traj-root <path>` (defaults to
`~/Library/Application Support/SAI/state/rl/scored/` and
`~/Library/Logs/SAI/trajectories/`).

## 5. Dry run script (`scripts/rl_dry_run.py`)

End-to-end smoke test with synthetic data. No external services required.

```sh
python -m scripts.rl_dry_run
```

Exercises all five phases:
1. Write 4 synthetic `RawTrajectory`s across 2 workflows
2. Score each via all three `HumanRewardScorer` signal sources
3. Assemble into `ScoredTrajectory` objects
4. Build preference pairs (`min_reward_gap=0.5`)
5. Export DPO, SFT, and scored-PPO JSONL to `/tmp/sai_rl_dry_run_*/`

Output can be fed directly into `sai_rl_report.py` for inspection:

```sh
DIR=$(ls -d /tmp/sai_rl_dry_run_* | tail -1)
python -m scripts.sai_rl_report \
  --scored-root "$DIR/scored" \
  --traj-root "$DIR/trajectories" \
  --pairs --tail 4
```
